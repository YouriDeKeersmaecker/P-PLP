# Database Layer

The `p_plp.db` package contains the database-facing part of the library.
It is responsible for:

- resolving datasource configuration
- creating SQLAlchemy engines
- creating and resetting the work schema
- reading OMOP and work tables
- providing reusable SQL fragments used by cohorts and feature engineering

## Folder structure

The database package is split by responsibility:

- [`config.py`](d:/School/BachelorProef/P-PLP/src/p_plp/db/config.py)
  - datasource and schema configuration
  - environment variable fallback
  - `SourceConfig`, `get_source_config`, `get_engine_config`
- [`engine.py`](d:/School/BachelorProef/P-PLP/src/p_plp/db/engine.py)
  - SQLAlchemy engine creation
  - attaches resolved source configuration to the engine
- [`schema.py`](d:/School/BachelorProef/P-PLP/src/p_plp/db/schema.py)
  - creates or resets the work schema
  - bootstraps required work tables such as `target_cohort` and `outcome_cohort`
- [`tables.py`](d:/School/BachelorProef/P-PLP/src/p_plp/db/tables.py)
  - table access helpers
  - `read_table`, `list_cdm_tables`, `list_work_tables`
  - `run_sql` and `fetch_df`
- [`sql.py`](d:/School/BachelorProef/P-PLP/src/p_plp/db/sql.py)
  - reusable SQL fragments
  - date arithmetic and age expressions

## Main public API

For most users, the main entry points are:

```python
from p_plp.db import get_engine, get_engine_config, list_cdm_tables, read_table
```

Example:

```python
from p_plp.db import get_engine, get_engine_config, read_table

engine = get_engine(source_name="eunomia")
person_df = read_table(engine, "person", schema=get_engine_config(engine).cdm_schema)
```

## Source configuration

The library supports two source names:

- `postgres`
- `eunomia`

The resolved configuration is represented by `SourceConfig`.
Configuration can be provided directly as function arguments, with environment variables used as a fallback.

Examples:

```python
from p_plp.db.config import get_source_config

config = get_source_config(
    source_name="postgres",
    database_url="postgresql+psycopg2://user:pw@localhost:5432/db",
    cdm_schema="synthealarge",
    work_schema="plp_work",
)
```

```python
config = get_source_config(
    source_name="eunomia",
    database_path=":memory:",
    cdm_schema="main",
    work_schema="plp_work",
)
```

## Engine creation

Use `get_engine()` as the single engine entry point:

```python
from p_plp.db import get_engine

engine = get_engine(source_name="postgres")
```

```python
engine = get_engine(source_name="eunomia", database_path=":memory:")
```

When the engine is created, the work schema is also ensured automatically.

## Schema management

The work schema stores the intermediate PLP tables generated by the library.
Examples include:

- `target_cohort`
- `outcome_cohort`
- `labels`
- feature tables
- `model_dataset`

Low-level helpers are available if needed:

```python
from p_plp.db.schema import ensure_schema, reset_schema
```

These are mostly internal plumbing, but they are useful in tests and debugging.

## Table helpers

The table helpers make it easier to inspect the database from Python.

Examples:

```python
from p_plp.db import get_engine, get_engine_config, list_cdm_tables, list_work_tables, read_table

engine = get_engine(source_name="eunomia", database_path=":memory:")

cdm_tables = list_cdm_tables(engine)
work_tables = list_work_tables(engine)
```

`read_table()` can read any table, and it also accepts a `CdmTable` enum value as the table name:

```python
from p_plp.db import CdmTable, get_engine_config, read_table

cdm_schema = get_engine_config(engine).cdm_schema

read_table(engine, CdmTable.PERSON, schema=cdm_schema)
read_table(engine, "person", schema=cdm_schema)
```

## SQL helpers

The SQL helpers are small reusable fragments used across the package.
They do not execute SQL by themselves.

Examples:

```python
from p_plp.db.sql import sql_date_add_days, sql_date_subtract_days, sql_age_expression
```

These helpers are used internally for:

- time-at-risk windows
- lookback windows
- age derivation in demographic features
